    Описаны все команды и аргументы командной строки Феникс Майнера

    Обратите внимание, что PhoenixMiner поддерживает большинство параметров командной строки Claymore майнера, поэтому вы можете использовать те же параметры командной строки, что и те, которые вы использовали бы с майнером Клеймора.

    Как использовать эти команды?

    Например первая команда -pool <host:port> указывается в .bat фале следующим образом:

    PhoenixMiner.exe -pool eu1.ethermine.org:4444

    Команда со знаком минус -pool остается, а вместо того, что в угловых скобках <host:port> мы подставляем сервер и порт пула. При этом, знаки <> мы не пишем.

    Команды для подключения к пулу

    -pool <host:port>
    Адрес пула (добавьте к имени хоста ssl:// для SSL соединения или http:// для соло-майнинга).
    Пример: PhoenixMiner.exe -pool ssl://eu1.ethermine.org:5555 {: .notice--success}
    -wal <wallet>
    Кошелек (некоторые пулы требуют добавления логина и / или воркера).
    -pass <password>
    Пароль (для большинства пулов он не требуется, используйте x качестве пароля, если не уверены, что на пуле есть пароль).
    -worker <name>
    Имя воркера (большинство пулов принимают его как название вашей фермы или кошелька, можно указать любое).

     

    Если указано <n>, то вместо <n> нужно подставить число без символов <>.
    -proto <n>
    Выбирает тип stratum протокола для пула ethash:
    1
    miner-proxy - спецификация stratum для майнинга в режиме прокси (например, для пула Coinotron)
    2
    eth-proxy - (например, для пулов ethermine, nanopool) - это значение по умолчанию, работает для большинства пулов.
    3
    qtminer (например, ethermine, ethpool)
    4
    EthereumStratum/1.0.0 (например для nicehash)
    5
    EthereumStratum/2.0.0
    -coin <coin>
    Монета, которую нужно использовать для devfee (комиссии), чтобы избежать переключения DAG файла. Доступные варианты:

    -stales <n>
    Отправлять stales (устаревшие решения) в пул: 1 - да (по умолчанию), 0 - нет
    -pool2 <host:port>
    Адрес второго отказоустойчивого пула. То же, что и -pool, но для отказоустойчивого пула.
    -wal2 <wallet>
    Кошелек отказоустойчивого пула (если он не указан, то первый -wal будет использоваться для отказоустойчивого пула).
    -pass2 <password>
    Пароль отказоустойчивого пула (если он отсутствует, то первая команда -pass будет использоваться и для пула отработки отказа).
    -worker2 <name>
    Имя воркера для запасного пула (если оно отсутствует, то -worker будет использоваться и для отказоустойчивого пула).
    -proto2 <n>
    Stratum протокол отказоустойчивого пула (если он отсутствует, то первая команда -proto будет использоваться также для пула отработки отказа).
    -coin2 <coin>
    Монета для devfee Ethash (если она отсутствует, то команда -coin также будет использована для пула отработки отказа).
    -stales2 <n>
    Отправлять устаревшие данные в пул отработки отказа: 1 - да (по умолчанию), 0 - нет
    -dpool <host:port>
    Адрес пула для дуал-майнинга.
    -dwal <wallet>
    Кошелек для дуал-майнинга.
    -dpass <password>
    Пароль для дуал-майнинга (для большинства пулов он не требуется, используйте x качестве пароля, если не уверены что пароль есть на пуле).
    -dworker <name>
    Имя второго воркера.
    -dcoin blake2s
    В настоящее время для двойного майнинга поддерживается только алгоритм Blake2s. Если вы хотите разместить все пулы двойного майнинга dpools.txt, вам нужно установить -dcoin blake2s в командной строке или в config.txt, чтобы майнер загружал пулы двойного майнинга из dpools.txt
    -dstales <n>
    Отправлять stales в пул двойного майнинга: 1 - да (по умолчанию), 0 - нет

    Общие параметры пулов

    -fret <n>
    Переключиться на следующий пул после N неудачных попыток подключения (по умолчанию: 3)
    -ftimeout <n>
    Повторное подключение, если новое задание не получено в течение n секунд (по умолчанию: 600)
    -ptimeout <n>
    Вернуться к основному пулу через n минут. По умолчанию этот параметр составляет 30 минут; установите значение 0, чтобы отключить автоматическое переключение обратно в основной пул.
    -retrydelay <n>
    Время ожидания перед повторным подключением в секундах (по умолчанию: 5)
    -gwtime <n>
    Период перепроверки для майнинга в режиме Solo/GetWork (по умолчанию: 200 мс)
    -rate <n>
    Сообщать о хешрейте в пул: 1 - да, 0 - нет (1 по умолчанию), 2 - (только для соло-майнинга) используйте альтернативное имя метода отчета eth_submitHashRate вместо eth_submitHashrate

    Тестовый режим

    Если указаны две команды через запятую, как в примере ниже, то это синонимы. Нужно использовать только одну.

    Например -bench [<n>],-benchmark [<n>]. Используем либо команду -bench [<n>] либо -benchmark [<n>].

    -bench [<n>],-benchmark [<n>]
    Режим тестирования, (необязательно) укажите эпоху DAG файла. Используйте эту команду, чтобы проверить свою майнинг ферму. Если вы укажете только эту -bench опцию, вы проведете тестирование алгоритма ethash. Если вы хотите протестировать дуал майнинг, воспользуйтесь опциями -bench <n> -dcoin blake2s. Если вы хотите протестировать алгоритм ProgPOW BCI, используйте параметры -bench <n> -coin bci

    Удаленный мониторинг и управление

    -cdm <n>
    Выбирает уровень поддержки удаленного мониторинга CDM:
    0
    отключен
    1
    только для чтения - это значение по умолчанию
    2
    полный (использовать только для безопасных соединений)
    -cdmport <порт>
    Установите порт удаленного мониторинга CDM (по умолчанию 3333). Вы также можете указать <ip_addr: port>, если у вас есть безопасное соединение VPN и вы хотите привязать к нему порт CDM.
    -cdmpass <пароль>
    Установите пароль удаленного мониторинга CDM
    -cdmrs
    Перезагрузите настройки, если config.txt редактируются / загружаются удаленно. Обратите внимание, что для изменения большинства параметров требуется перезапуск. В настоящее время следующие параметры могут быть изменены без перезагрузки: -mi , -gt , -sci , -clf , -nvf , -gpow , и большинство параметров аппаратного контроля ( -tt , -fcm , -fanmin , -fanmax , -powlim , -tmax , -ttli , -cclock , -cvddc , -mclock , -mvddc , -ppf , -straps , -vmt1 , -vmt2 , -vmt3 , -vmr ).

    Команды для управления майнингом

    -amd
    Используйте только карты AMD
    -acm
    Включите вычислительный режим AMD на поддерживаемых графических процессорах. Это эквивалентно нажатию y в консоли майнера.
    -nvidia
    Используйте только карты Nvidia
    -gpus <123 ..n>
    Используйте только указанные графические процессоры (если их больше 10, разделите индексы запятыми)
    -mi <n>
    Установите интенсивность майнинга (от 0 до 14; 12 по умолчанию для новых ядер). Вы можете указать эту опцию для каждого графического процессора.
    -gt <n>
    Установите параметр настройки GPU (от 6 до 400). Значение по умолчанию 15. Вы можете изменить параметры настройки в интерактивном режиме с + и - ключами в окне консоли шахтера. Вы можете указать эту опцию для каждого графического процессора. Если вы не укажете -gt или укажете значение 0, майнер будет использовать автонастройку для определения наилучшего значения GT. Обратите внимание, что когда графический процессор использует двойной майнинг, он игнорирует -gt значения и -sci вместо них использует .
    -sci <n>
    Установите интенсивность двойного майнинга (от 1 до 1000). Значение по умолчанию - 30. По мере увеличения значения -sci хешрейт вторичной монеты будет увеличиваться, но цена будет заключаться в более высоком энергопотреблении и / или более низком хешрейте ethash. Вы можете изменить этот параметр в интерактивном режиме с + и - ключами в окне консоли шахтера. Вы можете указать эту опцию для каждого графического процессора. Если вы установите -sci значение 0, майнер будет использовать автонастройку для определения наилучшего значения, пытаясь максимизировать хешрейт ethash независимо от хешрейта вторичной монеты.
    -clKernel <n>
    Тип ядра OpenCL: 0 - универсальное, 1 - оптимизированное, 2 - альтернативное, 3 - турбо (1 по умолчанию). Вы можете указать эту опцию для каждого графического процессора.
    -clgreen <n>
    Используйте энергоэффективные («зеленые») ядра (0: нет, 1: да; по умолчанию: 0). Вы можете указать эту опцию для каждого графического процессора. Обратите внимание, что вам нужно снова запустить автонастройку, так как оптимальные значения GT совершенно разные для зеленых ядер.
    -clNew <n>
    Используйте новые ядра AMD, если они поддерживаются (0: нет, 1: да; по умолчанию: 1). Вы можете указать эту опцию для каждого графического процессора.
    -clf <n>
    Синхронизация ядра AMD (0: никогда, 1: периодическая; 2: всегда; по умолчанию: 1). Вы можете указать эту опцию для каждого графического процессора.
    -nvKernel <n>
    Тип ядра Nvidia: 0 авто (по умолчанию), 1 старое (v1), 2 новее (v2), 3 последних (v3). Обратите внимание, что ядра v3 поддерживаются только графическими процессорами GTX10x0. Также обратите внимание, что двойной майнинг поддерживается только ядрами v2. Вы можете указать эту опцию для каждого графического процессора.
    -nvdo <n>
    Включите оптимизацию для конкретного драйвера Nvidia (0 - нет, по умолчанию; 1 - да). Попробуйте, -nvdo 1 если у вас нестабильно. Вы можете указать эту опцию для каждого графического процессора.
    -nvНовый <n>
    Используйте новые ядра Nvidia, если они поддерживаются (0: нет, 1: да; по умолчанию: 1). Вы можете указать эту опцию для каждого графического процессора.
    -nvf <n>
    Синхронизация ядра Nvidia (0: никогда, 1: периодическая; 2: всегда; 3: принудительно; по умолчанию: 1). Вы можете указать эту опцию для каждого графического процессора.
    -mode <n>
    Режим майнинга (0: двойной майнинг, если указаны двойные пулы; 1: ethash, даже если указаны двойные пулы). Вы можете указать эту опцию для каждого графического процессора.
    -gbase <n>
    Установите индекс первого графического процессора (0 или 1, по умолчанию: 1)
    -minRigSpeed ​​<n>
    Перезапустите майнер, если средняя скорость 5 минут ниже <n> MH / s.
    -eres <n>
    Выделяйте буферы DAG, достаточно большие на n эпох вперед (по умолчанию: 2), чтобы избежать выделения новых буферов при каждом переключении эпохи DAG, что должно улучшить стабильность переключения DAG. Вы можете указать эту опцию для каждого графического процессора.
    -daglim <n>
    (Только карты AMD Polaris, Baffin, Tonga или Fiji). Ограничьте размер DAG до <n> МБ, чтобы разрешить майнинг на картах 4 ГБ через несколько недель после эпохи 373 в Windows (или 381 в Linux). Возможные значения, кроме точного ограничения DAG, также равны 0 (отключить ограничение DAG), 1 (автоматический размер ограничения DAG, обычно 4023 МБ под Windows; это значение по умолчанию). Обратите внимание, что хешрейт будет значительно падать с каждой эпохой после того, как ограничение DAG вступит в силу. Если хешрейт падает слишком сильно (например, с 28 MH / s до 2-3 MH / s, вам нужно использовать более низкое значение, например -daglim 3996 или даже ниже.
    -dagrestart <n>
    Перезапустите майнер при выделении буфера для новой эпохи DAG. Возможные значения: 0 - никогда, 1 - всегда, 2 - авто (майнер решает в зависимости от версии драйвера). Это актуально для карт AMD 4 ГБ, у которых могут возникнуть проблемы с новыми эпохами DAG после эпохи 350.
    -lidag <n>
    Замедлите создание DAG, чтобы избежать сбоев при переключении эпох DAG (0–3, по умолчанию: 0 - самый быстрый, 3 - самый медленный). Вы можете указать эту опцию для каждого графического процессора.
    -gser <n>
    Сериализация создания DAG на нескольких графических процессорах (0 - без сериализации, все графические процессоры генерируют DAG одновременно, это значение по умолчанию; 1 - частичное перекрытие генерации DAG на каждом графическом процессоре; 2 - без перекрытия (каждый графический процессор ожидает, пока предыдущий не закончит создание DAG); 3-10 - задержка от 1 до 8 секунд после каждой генерации DAG GPU перед следующей)
    -gpureset <n>
    Полный сброс графического процессора при паузе (0 - нет, 1 - да; по умолчанию: нет, кроме 1080Ti). Вы можете указать эту опцию для каждого графического процессора.
    -rvram <n>
    Минимальный объем свободной видеопамяти в МБ (-1: не проверять; по умолчанию: 384 для Windows и 128 для Linux)
    -altinit
    Используйте альтернативный способ инициализации карт AMD, чтобы предотвратить сбои при запуске
    -wdog <n>
    Включить сторожевой таймер: 1 - да, 0 - нет (по умолчанию 1). Сторожевой таймер периодически проверяет, не зависает ли какой-либо из графических процессоров, и если да, перезапускает майнер (см. -rmode Параметр командной строки для режимов перезапуска).
    -wdtimeout <n>
    Тайм-аут сторожевого таймера (30 - 300; по умолчанию 45 секунд). Вы можете использовать этот параметр для увеличения тайм-аута сторожевого таймера по умолчанию на случай, если он перезапустит майнер без необходимости.
    -rmode <n>
    Выбирает режим перезапуска при сбое или зависании графического процессора:: 0: отключено - майнер будет выключен вместо перезапуска: 1: перезапустить с теми же параметрами командной строки - это значение по умолчанию: 2: перезагрузить (выключить майнер и выполнить reboot.bat )
    -log <n>
    Выбирает режим файла журнала:: 0: отключено - файл журнала записываться не будет: 1: записать файл журнала, но не отображать сообщения отладки на экране (по умолчанию): 2: записать файл журнала и показать сообщения отладки на экране
    -logfile <имя>
    Задайте имя файла журнала. Если вы поместите звездочку (*) в имя файла журнала, оно будет заменено текущей датой / временем для создания уникального имени при каждом запуске PhoenixMiner. Если в имени файла журнала нет звездочки, новые записи журнала будут добавлены в конец того же файла. Если вы хотите использовать один и тот же файл журнала, но его содержимое будет перезаписываться каждый раз при запуске майнера, поместите знак доллара ($) в имя файла журнала (например -logfile my_log.txt$ ).
    -logdir <путь>
    Задайте путь, по которому будут создаваться файлы журнала.
    -logsmaxsize <n>
    Максимальный размер файлов журнала в МБ. По умолчанию 200 МБ (используйте 0, чтобы отключить ограничение). При запуске, если размер файлов журнала превышает указанный предел, самые старые удаляются. Если вы используете один файл журнала (с помощью -logfile ), он обрезается, если он превышает лимит, и создается новый.
    -config <имя>
    Загрузите файл с параметрами конфигурации, которые будут добавлены к параметрам командной строки. Обратите внимание, что порядок важен. Например, если у нас есть config.txt файл, который содержит, -cclock 1000 и мы указываем командную строку -cclock 1100 -config config.txt , параметры из config.txt файла будут иметь приоритет, и результирующий -cclock будет иметь значение 1000. Если порядок обратный ( -config config.txt -cclock 1100 ), то второй параметр имеет приоритет, а в результате - cclock будет 1100. Обратите внимание, что -config разрешен только один вариант. Также обратите внимание, что если вы перезагрузите файл конфигурации с помощью c ключа или удаленного интерфейса, его параметры будут иметь приоритет над тем, что вы указали в командной строке.
    -timeout <n>
    Перезагрузите майнер в соответствии с -rmode через n минут.
    -pauseat <чч: мм>
    Поставьте майнер на паузу в формате чч: мм (время 24 часа). Вы можете указать несколько раз: -pauseat 6:00,12:00
    -resumeat <чч: мм>
    Возобновите майнер в hh :: mm (время в 24 часа). Вы можете указать несколько раз: -resumeat 8:00,22:00
    -gswin <n>
    Временное окно статистики графического процессора (5-30 секунд; по умолчанию: 15; используйте 0, чтобы вернуться к способу отображения мгновенной статистики до версии 2.8)
    -gsi <n>
    Интервал статистики скорости (5-30 секунд; по умолчанию: 5; используйте 0 для отключения). Подробная статистика по-прежнему отображается каждые 45 секунд и не зависит от -gsi значения.
    -astats <n>
    Показать расширенную статистику из веб-источников (0: нет; 1: да). По умолчанию курсы обмена монет обновляются каждые 4 часа, а сложность монет обновляется каждые 8 ​​часов. Вы можете увеличить эти периоды, указав, например -astats 12 , что период обновления будет увеличиваться до 12 и 24 часов соответственно.
    -gpow <n>
    Уменьшите использование графического процессора до n% от максимума (по умолчанию: 100). Если вы уже используете -mi 0 (или другое низкое значение), используйте -li вместо него. Вы можете указать эту опцию для каждого графического процессора.
    -li <n>
    Еще один способ снизить использование графического процессора. Большие значения n означают меньшую загрузку графического процессора; значение по умолчанию - 0. Вы можете указать эту опцию для каждого графического процессора.
    -resetoc
    Сбросьте настройки разгона HW при запуске
    -leaveoc
    Не сбрасывать настройки разгона при закрытии майнера

    -hwm <n>
    Частота мониторинга оборудования (одна настройка для всех карт, по умолчанию 1): 0 - нет мониторинга или управления HW, 1 - нормальный мониторинг, от 2 до 5 - реже мониторинг.
    -tt <n>
    Установите целевую температуру управления вентилятором (специальные значения: 0 - без управления вентилятором, отрицательное значение - фиксированная скорость вентилятора на n%)
    -hstats <n>
    Уровень аппаратного мониторинга: 0 - только температура и скорость вентилятора; 1 - температура, скорость вращения вентилятора и мощность; 2 - полный (включая частоту ядра / памяти, напряжения, P-состояния). По умолчанию 1.
    -pidle <n>
    Энергопотребление установки в холостом режиме в Вт. Будет добавлено к потребляемой мощности графического процессора при расчете общего энергопотребления установки.
    -ppf <n>
    Энергопотребление каждого графического процессора будет умножено на это значение, чтобы получить фактическое использование. Это значение выражается в процентах, поэтому, например, если графический процессор сообщает об энергопотреблении 100 Вт и вы указали -ppf 106, потребление энергии графическим процессором будет рассчитано как 100 * (106/100) = 106 Вт. Это позволяет вам исправить эффективность блоков питания и отдельных графических процессоров. Вы также можете указать это значение для каждого графического процессора отдельно.
    -prate <n>
    Цена на электроэнергию в долларах США за кВтч (например -prate 0.1 ). Если указано, майнер рассчитает суточную стоимость электроэнергии буровой установки.
    -fanmin <n>
    Установите минимальную скорость управления вентилятором в% (-1 по умолчанию)
    -fanmax <n>
    Установите максимальную скорость управления вентилятором в% (-1 по умолчанию)
    -fcm <n>
    Установите режим управления вентилятором (0 - автоматический, 1 - использовать управление вентилятором VBIOS, 2 - принудительное управление вентилятором; по умолчанию: 0)
    -fanidle <n>
    ( Только для Linux ) Установите скорость вращения вентилятора в% (-1 - автоматически, по умолчанию - 20).
    -fpwm <n>
    ( Только для Linux ) Fan PWM mode (0 - auto, 1 - direct, 2 - Polaris, 3 - Vega, 4 - Radeon VII, Navi; по умолчанию: 0)
    -tmax <n>
    Установите максимальную температуру управления вентилятором (0 по умолчанию). Если эта функция не поддерживается драйвером, -ttli вместо этого устанавливается та же температура.
    -powlim <n>
    Установите ограничение мощности графического процессора в% (от -75 до 75, 0 по умолчанию)
    -cclock <n>
    Установите частоту ядра графического процессора в МГц (0 по умолчанию). Для карт Nvidia используйте относительные значения (например, -300 или +400).
    -cvddc <n>
    Установите напряжение ядра GPU в мВ (0 по умолчанию)
    -mclock <n>
    Установите частоту памяти графического процессора в МГц (0 по умолчанию). Для карт Nvidia используйте относительные значения (например, -300 или +400).
    -mvddc <n>
    Установите напряжение памяти GPU в мВ (0 по умолчанию)
    -tstop <n>
    Приостановить графический процессор, когда температура> = n ° C (0 по умолчанию; т.е. выключено)
    -tstart <n>
    Возобновить работу графического процессора, когда температура <= n ° C (0 по умолчанию; т.е. выключено)
    -mt <n>
    Тайминги VRAM (только AMD под Windows): 0 - значения VBIOS по умолчанию; 1 - более быстрые тайминги; 2 - самые быстрые тайминги. По умолчанию - 0. Это полезно для майнинга с картами AMD без модификации VBIOS.
    -leavemt
    Не сбрасывайте уровень тайминга памяти ( -mt ) на 0 при закрытии.
    -ttli <n>
    Уменьшение использования графического процессора, когда температура графического процессора выше n ° C. Значение по умолчанию - 0, что означает, что использование не должно снижаться независимо от температуры графического процессора. Эта опция полезна, когда -tmax не работает. Если вы используете как -tt и -ttli варианты, температура -tt должна быть ниже , -ttli чтобы избежать удушения графических процессоров без использования вентиляторов , чтобы должным образом охлаждать их первым.
    -ремни <n>
    Уровень планки памяти (только карты Nvidia серии 10x0). Возможные значения: от 0 до 6. 0 является значением по умолчанию и использует тайминги по умолчанию из VBIOS. Каждый из которых соответствует уровню планки к предопределенной комбинации таймингов памяти ( -vmt1 , -vmt2 , -vmt3 , -vmr ). Уровень ремня 3 является самым быстрым из предустановленных уровней и может не работать на большинстве карт, 1 - самый медленный (но все же быстрее, чем время по умолчанию). Уровни ремешка с 4 по 6 такие же, как с 1 по 3, но с менее агрессивной частотой обновления (т. Е. Более низкими -vmr значениями).
    -ремни <n>
    Уровень планки памяти (только карты AMD Vega). Возможные значения: от 0 до 5. 0 является значением по умолчанию и использует тайминги по умолчанию из VBIOS. Каждому уровню ремешка соответствует заранее определенная комбинация таймингов памяти. Уровень ремешка 5 - самый быстрый уровень и может не работать на большинстве карт, 1 - самый медленный (но все же быстрее, чем время по умолчанию). Обратите внимание, что ремни для карт AMD являются экспериментальными и могут привести к сбоям или нестабильности. -vmt1 , -vmt2 , И -vmt3 параметры не имеют никакого влияния на картах AMD
    -vmt1 <n>
    Параметр времени памяти 1 (от 0 до 100, по умолчанию 0)
    -vmt2 <n>
    Параметр времени памяти 2 (от 0 до 100, по умолчанию 0)
    -vmt3 <n>
    Параметр времени памяти 3 (от 0 до 100, по умолчанию 0)
    -vmr <n>
    Частота обновления памяти (от 0 до 100, по умолчанию 0). Для карт AMD вы также можете использовать -rxboost
    -nvmem <n>
    Принудительное использование ремней на неподдерживаемых графических процессорах Nvidia (0 - не принудительно, 1 - GDDR5, 2 - GDDR5X). Убедитесь, что параметр соответствует типу памяти вашего графического процессора. Вы можете попробовать это, если ваша карта основана на Паскале, но при попытке использовать -straps или любой другой вариант синхронизации памяти карта отображается как «неподдерживаемая».
    -rxboost <n>
    Частота обновления памяти на картах AMD (0 - значение по умолчанию, 1 - предустановленное значение, которое должно работать на большинстве карт, от 2 до 100 - все более агрессивные настройки). Если вы хотите точно настроить значение, вы можете запустить майнер с -rxboost 1 , записать соответствующие -vmr значения, которые отображаются в файле журнала, а затем использовать -vmr вместо них с настроенными значениями.
    -vmdag <n>
    Сбросить ремни по умолчанию во время генерации DAG (по умолчанию: 1)
    -mcdag <n>
    Сбросьте тактовую частоту памяти графического процессора до значений по умолчанию во время создания DAG. Только Nvidia, по умолчанию: 0 (выключено). Это может позволить вам установить более высокий разгон памяти на ваших картах Nvidia, не рискуя испортить буфер DAG, что может привести к чрезмерному количеству устаревших общих ресурсов. В Linux эта опция будет выполнять daggen.sh сценарий (если он присутствует в текущем каталоге) для каждого GPU, передавая индекс GPU в качестве первого аргумента и идентификатор шины PCIE в качестве второго аргумента. Затем майнер будет ждать около 7 секунд перед запуском генерации DAG, чтобы дать сценарию достаточно времени для сброса разгона памяти.

    Варианты для каждй видеокарты отдельно

    Некоторые параметры PhoenixMiner могут предоставлять либо одинаковые настройки для всех графических процессоров, либо разные настройки для каждого из графических процессоров. Например, чтобы указать -gt значение для всех карт, которые вы должны написать, -gt 90 но если вы хотите указать другое значение GT для каждой из карт, используйте что-то вроде этого: -gt 20,15,40,90,90 для майнинг-фермы с пятью графическими процессорами. Это установит GT равным 20 для первого графического процессора, 15 для второго графического процессора и так далее. Если вы укажете меньше значений, чем у вас есть графические процессоры, остальные графические процессоры будут использовать значение по умолчанию для параметра. Вы также можете использовать другой, более гибкий способ указания разных значений для разных карточек. Лучше всего это пояснить на примере:-cclock *:1100,1-3:1090,4:1300 - здесь мы устанавливаем частоту ядра на 1100 МГц для всех карт, кроме карт с 1 по 3, на которых она установлена ​​на 1090 МГц, и карты 4 на 1300 МГц. Часть перед двоеточием (:) - это селектор, который выбирает графические процессоры, для которых применяется значение после двоеточия. Селектор может быть: <ul> <li>индекс одиночного графического процессора: например, 5:1000 устанавливает 1000 для 5-го графического процессора</li> <li>диапазон индексов графического процессора: например, 2-5:1200 устанавливает 1200 для графических процессоров 2, 3, 4 и 5</li> <li>звездочка, которая устанавливает значение для всех графических процессоров</li> <li>label amd или nvidia : например, amd:1090 устанавливает значение 1090 для всех карт AMD</li> <li>произвольная строка, которая начинается с буквы и может содержать буквы, числа и звездочки, что соответствует имени графического процессора, указанному PhoenixMiner. Пример: gtx*1070:+500 установит значение +500 для всех карт, в названии которых есть «gtx» и «1070», а между ними что-то есть. Это будет соответствовать «Nvidia GeForce GTX 1070», но не «Nvidia GeForce 1070».</li> </ul> Обратите внимание, что если данной карте соответствует более одного селектора, учитывается только последний. Пример: -cclock *:1100,1-4:1090,2:1300 установим карту 2 на 1300; карты 1,3 и 4 по 1090; а у остальных карт частота ядра 1100 МГц.
